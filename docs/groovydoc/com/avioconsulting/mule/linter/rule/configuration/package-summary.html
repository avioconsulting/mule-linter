Package com.avioconsulting.mule.linter.rule.configuration

  • Class Summary 
    Class Description
    ApiConsoleDisabledRule This rule checks that API Console has been removed or disabled (flow has attribute `initialState="stopped"`).
    CommentedCodeRule This rule checks that no code is found within comments inside mule configuration files.
    ComponentAttributesValueRule This rule checks that a component has a particular attribute, or a particular value on that attribute.
    ComponentCountRule This rule checks that a given component has not been used more than a specified number of times.
    ConfigFileNamingRule This rule checks that a config file matches a given case format.
    ConfigPlaceholderRule This rule checks that certain common config elements use String interpolation placeholcers (`${value}`) to point to provided properties rather than be specified statically in line.
    ConsecutiveLoggersCountRule This rule checks for multiple Logger components of the same level being used back to back.
    DisplayNameRule This rule ensures that developers are taking full advantage of display name when considering certain components in Anypoint Studio.
    FlowSubflowNamingRule This rule ensures that `flow` and `sub-flow` names follow a given case format.
    GlobalConfigExistsRule This rule checks that a global mule config exists, and that the expected configuration elements are not in other files.
    GlobalConfigNoFlowsRule This rule checks that a global mule configuration file does not contain flows.
    LoggerAttributesRule This rule checks that the Loggers have required attributes.
    LoggerCategoryExistsRule This rule checks that the category attribute is present on all loggers.
    LoggerMessageContentsRule This rule checks that the contents of a logger do not match a given Regular Expression.
    LoggerMessageExistsRule This rule checks that the message attribute is present on all loggers.
    MuleConfigFlowLimitRule This rule checks that the Mule Configuration file does not contain too many flows or subflows.
    OnErrorLogExceptionRule This rule checks that the logException rule is enabled for all `on-error-continue` and `on-error-propagate` components.
    UntilSuccessfulRule This rule checks that the until-successful component has not been used.
    UnusedFlowRule This rule checks that all flows or sub-flows are currently in use by this application.
